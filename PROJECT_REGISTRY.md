# Project Registry

This file tracks the major projects in the Super ecosystem.

Each serious project should exist as its own clean GitHub repository and be linked from here.

The `Super` repository acts as the command-center.  
Individual project repositories contain the actual source code.

---

## Registry Rules

Each project entry should include:

- Project name
- Repository link
- Tier
- Domain
- Purpose
- Standard expected
- Current repository status
- Documentation status
- Future integration role

Projects are grouped into:

- Phase Projects
- Tier 1 Flagship Projects
- Tier 2 Supporting Projects
- Tier 3 Learning Projects

# Registry Maintenance Rule

This file does not need to be updated after every small learning step.

Update it only when:

- A new serious project repository is created
- A project is completed
- A project is professionalized
- A major project changes tier
- A major architecture or integration decision changes

The goal is to keep the registry stable, high-level, and useful.
